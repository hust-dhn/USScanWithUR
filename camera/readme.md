# 相机标定（左相机与右相机）

## 介绍

此项目用于执行相机标定，分为 **左相机** 和 **右相机** 两个部分。两个相机的标定将对应同一个机器人位姿文件，以便实现相机与机器人之间的准确配准。

## 目录结构

项目目录结构如下：


- **camera/**：包含用于标定。
- **lc_imgs/**：左相机的标定图像。图片命名格式是1.jpg/1.png
- **rc_imgs/**：右相机的标定图像。图片命名格式是1.jpg/1.png
- **calibration.py**：执行标定的 Python 脚本。
- **config_ic.yaml**：左相机的标定配置文件。
- **config_rc.yaml**：右相机的标定配置文件。
- **robot_pos.txt**：机器人末端的位姿文件，两个相机共享此文件。

---
## 标定流程

1. **准备相机图像**：
    * 将左相机的标定图像（如 `1.jpg`, `2.jpg` 或 `1.png`, `2.png`）放入 `lc_imgs/` 文件夹。
    * 将右相机的标定图像（如 `1.jpg`, `2.jpg` 或 `1.png`, `2.png`）放入 `rc_imgs/` 文件夹。

2. **编辑配置文件**：
    * 根据实际情况修改 `config_ic.yaml` 和 `config_rc.yaml` 中的参数
    * 确保图像文件夹路径和图像格式（`.jpg` 或 `.png`）正确。

3. **运行标定脚本**：
    * 使用 `calibration.py` 脚本进行标定。
    
4. **检查结果**：
    * 脚本运行后，会输出标定结果，并将结果保存到文件中。
    
    **输出的变换矩阵**：
    - 每个相机将输出一个 4x4 的 **变换矩阵**，该矩阵表示相机坐标系到机器人TCP坐标系的转换。变换矩阵包括 **旋转矩阵** 和 **位移向量**。
    - 变换矩阵的一般格式为：
    
    ```
    [R11, R12, R13, T1]
    [R21, R22, R23, T2]
    [R31, R32, R33, T3]
    [0,   0,   0,   1]
    ```
    - 其中，`R` 是旋转部分，`T` 是位移向量部分。矩阵表示了从相机坐标系到机器人TCP坐标系的转换关系。
    - 输出的变换矩阵将保存在指定的文件中（例如 `output_transformation_matrix.txt`），您可以根据需要使用这些数据进行进一步的计算或分析。
