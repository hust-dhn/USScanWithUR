# ur10e运动学分析

## 介绍

此项目用于对于工业六轴机器人的公式化正/逆运动学计算原理分析代码。

## 目录结构

项目目录结构如下：


- **kinematics/**：主要用于打通ur10e机器人的运动学控制部分
    **ik/**：针对UR10e型号的工业机器人进行逆运动学计算，是一个批量处理机器人逆运动学的工具，专门用于将一系列机器人末端位姿转换为对应的关节角度。
    **ur10e_kine/**：该代码用于UR10e机器人运动学计算的Python类，实现正、逆运动学功能
    **testify/**：用于验证机器人相机标定矩阵是否正确

---
## ur10e_kine.py/ 的函数（可直接调用）

1. **FK**：
    * 工具末端相对于机器人基坐标系的正运动学函数，包含了工具末端相对于机器人末端法兰坐标系变换矩阵

2. **FK_wo_end**：
    * 机器人末端法兰中心相对于机器人基坐标系的正运动学函数，不包含工具末端相对于机器人末端法兰坐标系的变换矩阵

3. **IK**：
    * 工具末端相对于机器人基坐标系的逆运动学函数，依照与初始位置最近原则选取关节角，包含了工具末端相对于机器人末端法兰坐标系变换矩阵
    
4. **IK_wo_end**：
    * 机器人末端法兰中心相对于机器人基坐标系的逆运动学函数，不包含工具末端相对于机器人末端法兰坐标系的变换矩阵
    
## IK.py/ 的使用流程

    初始化UR10e机器人的运动学模型

    从文件读取末端位姿（位置 + 姿态）

    对每个位姿：

        转换为4×4齐次变换矩阵

        计算逆运动学解

        保持解的连续性

    将关节角保存到输出文件

## testify.py/ 的检验标定矩阵流程

    首先，阅读/camera/readme.md_camera，运行/camera/get_imgs_and_poses.py，得到10个机器人末端法兰位姿与相机棋盘格图片；
    其次，将工具末端中心点移动至棋盘格的1*1点处；
    最后，运行testify.py，得到量化的精度误差。
